be modified directly after creation.

value as fixed. Any apparent modification creates a new string value
instead of changing the original string.

# Day 5 --- Learning Conditional Logic in JavaScript

**Date:** June 20, 2026

The focus of Day 5 was understanding how programs make decisions using
conditional statements.

## JavaScript Learning

### Conditional Statements

Learned how JavaScript can execute different blocks of code depending on
whether a condition evaluates to true or false.

Topics covered:

### if Statement

The basic decision-making structure.

Example concept:

``` javascript
if (condition) {
    // execute code
}
```

### if...else Statement

Used when there are two possible outcomes.

Example concept:

``` javascript
if (condition) {
    // true block
} else {
    // false block
}
```

### else if Statement

Used when multiple conditions need to be checked.

Example concept:

``` javascript
if (condition1) {

} else if (condition2) {

} else {

}
```

### Comparison Operators

Learned how operators help create conditions:

-   Greater than (`>`)
-   Less than (`<`)
-   Equal to (`===`)
-   Not equal (`!==`)
-   Greater than or equal to (`>=`)
-   Less than or equal to (`<=`)

### Logical Operators

Explored combining multiple conditions using:

-   AND (`&&`)
-   OR (`||`)
-   NOT (`!`)

## Key Understanding

Conditional statements are the foundation of program logic. They allow
software to respond differently based on inputs, user actions, or
changing situations.

Understanding conditionals is an important step towards building
interactive applications and intelligent systems.
